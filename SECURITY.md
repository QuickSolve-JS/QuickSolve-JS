# Security Policy

## Supported Versions

We actively provide security updates for the following versions of **QuickSolve-JS**:

| Version | Supported          |
| ------- | ------------------ |
| v1.0.1  | ✅ Yes              |
| < v1.0  | ❌ No               |

## Reporting a Vulnerability

If you discover a security vulnerability within QuickSolve-JS, please help us keep the project secure by reporting it responsibly. 

**Please do not open a public GitHub Issue for security-related bugs.**

Instead, please follow these steps:
1. Email us directly at [nidurangajayarathna@gmail.com](mailto:nidurangajayarathna@gmail.com).
2. Include a detailed description of the vulnerability.
3. Provide steps to reproduce the issue (PoC).

### Our Response Process
- We will acknowledge receipt of your report within **48 hours**.
- We will provide a timeline for a fix and keep you updated on the progress.
- Once the fix is verified, we will release a new version and give you credit for the discovery (if desired).

## Security Best Practices for Users
When using this package for financial calculations:
- Always use the latest stable version.
- Ensure your PHP environment has the `BCMath` extension enabled for high-precision calculations.
- Validate all input values before passing them to the calculation engine.
